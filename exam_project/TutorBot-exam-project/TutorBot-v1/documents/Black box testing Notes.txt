1 Equivalence Partitioning:

	Input Partions:

 	similar inputs can be grouped together (positive integers, alphabetic characters)
	any partitions calue can used for testing (often a middle value)
	numbers: zero is a special case that consitute a partition of its own
	input partions (based on input values)
	
example: program that accepts integers between -10000 and 10000
	should be split in valid and non valid partitions:
	valid:
	integer between -10000 and -1 (inclusive)
	for 0
	integer between 1 and 10000 (inclusive)
	non-valid:
	below -10000
	above 10000
	non-integer numbers (edge chase)
	non-numeric characters(edge chase)

	this would translate into 3 "positive" tests(partitions) and 4 "negative" tests (partitions)

Always remember edge chases!

	Output Partitions:

	These are based on outputs!
	Depending on the input, we expect different Outputs. Kinda the same as input partions, find the inputs that give the expected outputs.
