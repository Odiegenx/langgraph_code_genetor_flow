&nbsp;	**Trait:** How you want the LLM to think when its answering questions,

&nbsp;	You could give it roles or specify a specifig behavior

&nbsp;	

&nbsp;	**Task:** Be as specific as possible



&nbsp;	**Tone:** Style of language that the LLM should use

&nbsp;		- clear language

&nbsp;		- straigtforward language

&nbsp;		- stuff to avoid (fx hype language)

&nbsp;		- use a formal or proffesionel language

All these T's are giving the LLM a context for who the answer is for and how it should construct said answer.
