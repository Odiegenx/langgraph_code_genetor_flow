# Exam Readiness Gap Analysis

## Current Situation

The exam requirements are different from the mandatory assignment requirements.

The mandatory project focused on:

- comparing workflow frameworks
- building a LangGraph-based multi-agent coding workflow
- using local/Ollama-compatible endpoints
- producing generated software artifacts

The exam project must instead deliver a working end-user software prototype that includes:

- a local LLM in the delivered application
- RAG as part of the answer generation flow
- visible use of the 4T prompt engineering approach
- clear MVP scope
- setup guide
- technical documentation
- synopsis of maximum 5 pages

## Main Gap

The current Ollama Chat UI is not enough for the exam because it does not clearly implement RAG.

It is currently closer to:

```text
User prompt -> local Ollama model -> answer
```

The exam requires:

```text
User question -> retrieve relevant source material -> local LLM answers using retrieved context
```

## Recommended Exam Project Direction

The safest project direction is:

```text
Local RAG Study Assistant
```

The application should let a user ask questions about a small local knowledge base, for example:

- course notes
- assignment requirements
- project documentation
- selected markdown/text/pdf-derived documents

The answer should be generated by a local Ollama model and grounded in retrieved source chunks.

## Recommended MVP

The MVP should include:

- a small web UI
- a local document folder
- a simple ingestion/indexing step
- local embeddings or a simple retrieval strategy
- a chat/question input
- retrieved context shown or cited in the answer
- local Ollama answer generation
- 4T-based prompt template
- setup guide
- short technical documentation

## Suggested Architecture

```text
documents/
  source markdown/text files

backend or local script
  loads documents
  chunks text
  creates/searches embeddings or keyword index
  sends retrieved context to local LLM

frontend
  user asks a question
  displays answer
  displays retrieved sources

local Ollama
  generates final answer from prompt + retrieved context
```

## 4T Prompt Design

The delivered solution should contain an explicit prompt template using:

- Traits: helpful, precise, source-grounded assistant
- Task: answer the user's question using retrieved context
- Tone: concise, educational, honest about uncertainty
- Target: student/user trying to understand the selected documents

The prompt should instruct the model to:

- answer from retrieved context
- cite or name the used source chunks
- say when the context is insufficient
- avoid inventing facts not present in the retrieved material
